# Сопроводительная записка для заказчика

**Проект:** VSecureTransfer — защищённая передача одного видеофайла по TCP (шифрование AES-256-GCM, SHA-256, RSA-PSS, RSA-OAEP).  
**Репозиторий:** исходный код + скрипты сборки и приёмочных проверок. **Готовые секретные ключи в поставку не входят** — их генерирует получатель/разработчик локально.

---

## 1. Что лежит в `keys/` и что нельзя коммитить

После команды **`./scripts/gen_keys.sh`** в каталоге **`keys/`** появляются четыре PEM-файла:

| Файл | Назначение |
|------|------------|
| `sender_sign_priv.pem` | Закрытый ключ **отправителя** (подпись пакета) |
| `sender_sign_pub.pem` | Открытый ключ отправителя — у **получателя** для проверки подписи |
| `receiver_wrap_pub.pem` | Открытый ключ **получателя** (обёртка AES-ключа OAEP) — у **отправителя** |
| `receiver_wrap_priv.pem` | Закрытый ключ получателя — только у получателя |

**Не коммить и не пересылать по открытым каналам:** `*_priv.pem`, любые копии `keys/`, сгенерированные тестовые выходы с чувствительными путями. В `.gitignore` уже указаны `keys/` и `*.pem`.

Для **демонстрации** заказчику достаточно один раз сгенерировать ключи на своей машине и **не** публиковать приватные части.

## 2. Порядок действий (приёмка и демо)

1. **Клонировать** репозиторий, перейти в корень проекта (`VSecureTransfer/`).
2. **Установить зависимости:** компилятор C++17, OpenSSL 3, `cmake` (для юнит-тестов), `python3`, `bash` — см. основной [`README.md`](../README.md).
3. **Сгенерировать ключи:** `./scripts/gen_keys.sh`
4. **Собрать:**
   - CMake (рекомендуется, включает Catch2):  
     `cmake -S . -B build -DCMAKE_BUILD_TYPE=Release -DVSECURE_BUILD_TESTS=ON` → `cmake --build build -j`
   - или `make -j` (macOS/Linux с OpenSSL в типичном пути Homebrew — см. `Makefile`).
5. **Полная автоматическая проверка:** `./scripts/qa_full.sh`  
   Ожидаемый финал: строка **`=== Все проверки QA пройдены ===`** и ненулевой код только при реально найденной ошибке (должен быть **0**).

**Ручное демо (два терминала):**

1. Терминал A — получатель (порт **произвольный свободный**, пример **9000**):

   ```bash
   ./build/vsecure_receiver --port 9000 --out-dir ./out \
     --sender-pub keys/sender_sign_pub.pem \
     --recv-priv keys/receiver_wrap_priv.pem
   ```

2. Терминал B — отправитель (хост **127.0.0.1** или **localhost**; тот же порт):

   ```bash
   ./build/vsecure_sender --file ./video.mp4 --host 127.0.0.1 --port 9000 \
     --sign-key keys/sender_sign_priv.pem \
     --recv-pub keys/receiver_wrap_pub.pem
   ```

**Пути:** `--out-dir` — каталог, куда получатель сохранит файл (создаётся при необходимости); опционально **`--seen-file ПУТЬ`** — файл учёта `message_id` (по умолчанию `./out/.vsecure_seen`).

## 3. Платформы: Linux, macOS, Windows

| Среда | Статус |
|--------|--------|
| **Linux** (g++/clang, OpenSSL 3) | Нативная сборка CMake/Makefile, CI на `ubuntu-latest`. |
| **macOS** (Apple Clang, OpenSSL из Homebrew) | Нативная сборка; при необходимости `brew install cmake`. |
| **Windows** | Исходники ориентированы на **POSIX** (`poll`, `mmap`, `mkstemp`, сокеты BSD-стиля). **Нативная сборка под MSVC в этой поставке не поддерживается** без отдельного портирования. |

**Как запускать на Windows «как на всех ОС»:**

1. **Рекомендуется — Docker Desktop** (или другой Docker с Linux-контейнерами). Из корня репозитория на **Windows/macOS/Linux**:

   ```bash
   docker build -t vsecure-transfer .
   docker run --rm -it vsecure-transfer bash -lc 'cd /app && ./scripts/gen_keys.sh && ./scripts/qa_full.sh'
   ```

   Ожидается финал **`=== Все проверки QA пройдены ===`**. Сборка внутри образа уже выполнена (`Dockerfile`); `qa_full` дополнительно проверяет окружение.

2. **Альтернатива — WSL2 (Ubuntu):** установить WSL2, открыть терминал Ubuntu, клонировать репозиторий в файловую систему Linux (`~/...`), выполнить п.2–5 раздела «Порядок действий» этого документа.

ТЗ не требует именно «exe под Windows»; требуется рабочая консольная демонстрация в локальной сети. **Контейнер или WSL2** дают тот же бинарный результат, что и Linux, без переписывания POSIX-транспорта под WinAPI.
